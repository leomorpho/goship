# TKT-584: Add Playwright test proving the full auth golden flow: register → login → protected route → logout

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-584` before committing. -->

## Description
Add a Playwright browser test that proves the complete auth user journey works end-to-end in a real browser: (1) visit /register, fill form, submit → lands on dashboard. (2) log out → redirected to login. (3) visit protected route unauthenticated → redirected to login. (4) log in → lands on protected route. This is the most critical browser confidence gate for the framework.

Likely paths:
- tests/playwright/
- e2e/

Verify commands:
- npm run test:e2e
- npx playwright test
- docket validate $TICKET_ID

Out of scope:
- OAuth/social login flows.
- Password reset flows.
- Visual regression testing.

## Acceptance Criteria
- [ ] Playwright test visits /register
- [ ] submits form
- [ ] and lands on dashboard.
- [ ] Test logs out and verifies redirect to login.
- [ ] Test visits protected route unauthenticated and verifies redirect to login
- [ ] then logs in successfully.
