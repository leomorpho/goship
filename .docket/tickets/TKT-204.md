# TKT-204: G4-01: Implement ship destroy MVP reverse scaffolding

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-204` before committing. -->

## Description
`ship destroy` is still missing as the reverse path for generated framework artifacts. This ticket is
for the MVP command surface: remove generator-managed files and registrations deterministically,
refuse destructive deletes outside generated seams, and document the exact safety boundary.

Likely paths:
- .docket/tickets/TKT-204.md
- tools/cli/ship/internal/commands/destroy.go
- tools/cli/ship/internal/commands/destroy_test.go
- tools/cli/ship/internal/generator/*
- docs/roadmap/01-framework-plan.md
- docs/reference/01-cli.md
- docs/architecture/06-known-gaps-and-risks.md

Verify commands:
- go test ./tools/cli/ship/internal/commands -run 'Test.*Destroy' -count=1
- docket validate TKT-204

Out of scope:
- Removing user-authored files with no generator ownership signal.
- Full reverse support for every generator or battery in one step.

## Acceptance Criteria
- [ ] `ship destroy` removes generator-owned files and registrations for the documented MVP targets in a deterministic order.
- [ ] The command refuses to delete paths that are not recognized as generator-managed and reports the skipped reasons clearly.
- [ ] Command tests and CLI docs cover the supported destroy targets and the explicit safety boundary.
