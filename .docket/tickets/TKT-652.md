# TKT-652: upgrade ship generators from raw TODO placeholders to intentional scaffolds

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-652` before committing. -->

## Description
Several ship generators still emit raw TODO strings into generated app code or generated command skeletons. Some placeholder guidance is acceptable, but the current output reads like unfinished framework internals instead of deliberate scaffolding.

Confirmed examples:
- tools/cli/ship/internal/generators/command.go returns "TODO: describe this command."
- tools/cli/ship/internal/generators/resource.go and controller.go inject TODO comments and placeholder page text into generated resources
- tools/cli/ship/internal/generators/job.go emits TODO fields and TODO handler behavior
- tools/cli/ship/internal/commands/destroy.go depends on those TODO page markers to recognize generated files

Fix:
1. Replace raw TODO prose with intentional scaffold markers or minimal working defaults
2. Ensure generated code communicates extension points cleanly without looking unfinished
3. Update destroy/cleanroom tests so generator-managed artifacts are still identifiable without brittle TODO text matching
4. Keep any remaining placeholder conventions explicit and documented

Likely paths:
- tools/cli/ship/internal/generators/command.go
- tools/cli/ship/internal/generators/resource.go
- tools/cli/ship/internal/generators/controller.go
- tools/cli/ship/internal/generators/job.go
- tools/cli/ship/internal/commands/destroy.go
- related generator/cleanroom tests

Out of scope: fully implementing app-specific domain logic in generated stubs.

Verify commands:
- go test ./tools/cli/ship/internal/...
- tools/scripts/test-cleanroom.sh

## Acceptance Criteria
- [x] ship-generated command/resource/job scaffolds no longer expose raw unfinished TODO prose by default : implemented + verified in this branch
- [x] generated files still make extension points obvious to app developers : implemented + verified in this branch
- [x] destroy and generator tests identify managed artifacts without depending on brittle TODO strings : implemented + verified in this branch
- [x] go test ./tools/cli/ship/internal/... passes : implemented + verified in this branch
- [x] tools/scripts/test-cleanroom.sh passes : implemented + verified in this branch
