# TKT-159: DX023: Make Renders/Route comment detection block-aware (order-agnostic)

## Description
Current blocker: DX023 currently flags 80 templates that already have Renders comments because the checker only inspects the nearest previous non-empty line.

Goal: strengthen parser to understand comment blocks and avoid false positives when Route(s) and Renders are both present.

Design requirement: preserve strictness (Renders still required) while removing order-dependent fragility.

## Acceptance Criteria
- [ ] Test-first: add failing doctor tests for templ comment blocks where Route(s) and Renders appear in either order above exported templ declarations.
- [ ] hasRendersComment logic accepts contiguous comment blocks above templ declarations, not only immediate single-line adjacency.
- [ ] One blank separator line between comment block and templ declaration is explicitly covered by tests and policy decision.
- [ ] ship doctor no longer emits DX023 false positives for templates that already include a Renders comment in the adjacent comment block.
- [ ] Run and pass: go test ./tools/cli/ship/internal/policies -run Renders
