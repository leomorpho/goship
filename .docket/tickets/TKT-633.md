# TKT-633: Add an API-only fresh-app confidence suite that proves ship new --api boots, verifies, and exposes canonical JSON/auth surfaces.

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-633` before committing. -->

## Description
Objective: Add an API-only fresh-app confidence suite that proves `ship new --api` boots, verifies, and exposes canonical JSON/auth surfaces.

Context:
The default starter already has a fresh-app confidence story. If GoShip supports an API-only backend as a blessed path, that mode needs the same level of executable proof: generate the app, migrate it, boot it, verify it, and assert the canonical JSON and auth surfaces behave as documented.

Rails/Laravel DX bar:
A supported starter mode should feel first-party and production-credible on first boot, with tests proving the claim instead of docs implying it.

Likely paths:
- tools/cli/ship/internal/commands/project_new_integration_test.go
- tools/cli/ship/internal/commands/verify_test.go
- tools/cli/ship/tests/integration/
- docs/guides/08-building-an-api.md
- docs/reference/01-cli.md

Likely symbols:
- `ship new --api`
- API-only starter mode
- JSON response helpers
- auth/session/CSRF contract

Tests to add or update:
- temp-dir generation test for `ship new --api`
- verify/integration coverage for API-only boot and canonical JSON routes
- fixture coverage for auth/session behavior in the blessed API-only mode

Verify commands:
- docket validate TKT-633
- go test ./tools/cli/ship/internal/commands ./tools/cli/ship/tests/integration -count=1

Out of scope:
- Building the SvelteKit reference app itself.
- Supporting arbitrary frontend frameworks.
- Hosted deployment automation.

## Acceptance Criteria
- [x] A fresh API-only app generated by ship new --api boots, migrates, verifies, and passes canonical JSON/auth integration checks. : Added API-mode fresh-app confidence integration test that generates with `ship new --api`, runs db:migrate + verify fast, boots cmd/web, and asserts canonical JSON/auth route responses.
- [x] The API-only starter path has executable confidence coverage comparable to the default starter, with regression tests wired into the supported verification loop. : Wired regression coverage into supported loops via commands tests plus package-level integration test path used by verification command.
