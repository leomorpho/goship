# TKT-561: Remove conflicting ad hoc styling patterns from first-party modules and prove modules render with shared tokens.

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-561` before committing. -->

## Description
Objective: Remove conflicting ad hoc styling patterns from first-party modules and prove modules render with shared tokens.

Context:
This is the shared rendering surface between templ and islands. Keep recipes, tokens, and layout primitives consistent so a feature can move between server-rendered and island-rendered without inventing a second design system.

Rails/Laravel DX bar:
- Prefer one obvious convention, delete fallback paths, and keep the generated or framework-owned surface beautiful enough to feel like Rails or Laravel.

Likely files:
- framework/web/ui/page.go
- framework/web/ui/form.go
- framework/web/viewmodels/
- frontend/islands/
- styles/styles.css
- styles/tailwind_components.css

Likely symbols:
- Page
- FormSubmission
- LandingPage

Tests to add or update:
- framework/web/ui/*_test.go
- browser parity tests in e2e/ or command contracts

Verify commands:
- docket validate TKT-561
- go test ./framework/web/ui ./framework/web/viewmodels -count=1

Out of scope:
- Broad refactors outside the scoped ownership surface for this ticket.
- Writing roadmap docs that are not required to keep code and tests aligned.
- Taking shortcuts that preserve multiple competing conventions instead of choosing one clear path.
