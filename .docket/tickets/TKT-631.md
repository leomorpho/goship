# TKT-631: Add a cold-start agent eval task pack that proves weaker models find and use the right GoShip surfaces.

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-631` before committing. -->

## Description
Objective: Add a cold-start agent eval task pack that proves weaker models find and use the right GoShip surfaces.

Context:
Contract tests prove correctness, but they do not prove LLM usability. GoShip needs a small, representative task pack that starts from a cold prompt and measures whether an agent picks the right context/tooling path, avoids dead ends, and completes the task successfully.

Rails/Laravel DX bar:
The framework should be easy enough that a weaker model can start cold, find the right path quickly, and finish common tasks without repo archaeology or repeated human steering.

Likely paths:
- tools/
- docs/guides/01-ai-agent-guide.md
- docs/reference/01-cli.md
- docs/reference/02-mcp.md
- .github/workflows/test.yml

Likely symbols:
- cold-start task pack
- agent eval scorer
- success rate
- first-surface selection

Tests to add or update:
- deterministic eval fixtures or task definitions
- scoring/reporting tests for the eval harness
- docs describing the supported eval pack and what it measures

Verify commands:
- docket validate TKT-631
- go test ./... -count=1

Out of scope:
- Trying to benchmark every model on the market.
- Replacing the existing test suite with agent evals.
- Product analytics or hosted telemetry work.

## Acceptance Criteria
- [x] A representative cold-start task pack measures whether agents pick the right context/tooling path and complete common GoShip tasks. : Added representative cold-start task pack fixture plus evaluator that checks first-surface/tooling-path selection and task completion outcomes.
