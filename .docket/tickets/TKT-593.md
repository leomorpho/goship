# TKT-593: Epic: Code-defined feature flags with auto-sync and capabilities API

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-593` before committing. -->

## Description
Top-level epic for introducing a code-first, four-layer feature flag system to goship.

**The four layers:**
- **Layer A** (Source of Truth): Flags are typed Go constants (`FlagKey`) with a registered `FlagDefinition` (description, default). No more raw strings.
- **Layer B** (Syncer): On app startup, each registered flag is upserted into the DB — creating it if missing, refreshing description, but never touching the `enabled` state an admin set.
- **Layer C** (Evaluator): `Service.Enabled()` gains a code-default fallback so a DB outage never causes a panic or wrong answer.
- **Layer D** (Capabilities API): `GET /api/me/capabilities` returns a flat `map[string]bool` for the authenticated user — frontend never touches the DB directly.

**Admin UI enhancements:**
- Flags list shows the constant key and hardcoded default alongside live DB state.
- Orphaned DB rows (flag in DB but not in registry) are visually marked.
