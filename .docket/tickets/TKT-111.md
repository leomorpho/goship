# TKT-111: A09 - Backup Capability Contract And S3-Compatible Providers

## Description
**Source:** `docs/architecture/09-standalone-and-managed-mode.md`
**Roadmap Group:** Framework capability
**Original Status:** `[ ] todo`

**Status:** `[ ] todo`
**Files:** `framework/backup/` (new), `config/config.go`, `framework/core/interfaces.go`, tests under `framework/backup/`

**Context:**
Backup capability must live in GoShip so standalone apps can use it directly, while managed apps
can still be orchestrated by an external control plane. The contract should support SQLite first
and leave room for Postgres later.

**What to do:**
1. Define backup and restore driver interfaces in GoShip.
2. Define a typed backup manifest format with DB mode, schema version, checksum, and storage location.
3. Add S3-compatible provider configuration as the storage target boundary.
4. Implement the first SQLite-oriented backup path under the new contract.
5. Add tests for manifest generation and provider config validation.

**Done when:** GoShip has a standalone backup contract with manifest metadata and S3-compatible
provider support, and SQLite backups can be described through that contract without any control-plane dependency.
