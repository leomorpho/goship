# TKT-603: Add --ui flag to ship new and thread UIProvider through NewProjectOptions

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-603` before committing. -->

## Description
Objective: Make ship new provider-aware by accepting a --ui flag that records the chosen provider in the scaffold options.

Context: This is the CLI surface that lets a developer declare their UI choice at project init time. Depends on TKT-602 for the config.uiprovider type and constants.

Likely files:
- tools/cli/ship/internal/commands/project_new.go
  - Add UIProvider string field to NewProjectOptions
  - ParseNewArgs: handle --ui <provider> flag, validate against UIProviderFranken/UIProviderDaisy/UIProviderBare, return error on unknown value
  - Default UIProvider to UIProviderFranken when flag is absent
- tools/cli/ship/internal/commands/project_new_test.go
  - TestParseNewUIFlag: table-driven over valid providers and invalid values

Likely symbols:
- NewProjectOptions.UIProvider (new field)
- ParseNewArgs (updated)

Tests to add or update:
- tools/cli/ship/internal/commands/project_new_test.go: TestParseNewUIFlag

Verify commands:
- docket validate TKT-<id>
- go test ./tools/cli/ship/internal/commands/... -run TestParseNew -count=1

Out of scope:
- Writing the scaffold files themselves (that is TKT next).
- Any changes to config package.

## Acceptance Criteria
- [ ] NewProjectOptions.UIProvider is populated from --ui flag or defaults to franken
