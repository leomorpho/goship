# TKT-578: Add Otter in-memory cache adapter and wire it as the default dev/single-binary cache backend

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-578` before committing. -->

## Description
Add an Otter in-memory cache adapter so GoShip can run without Redis in standalone/single-binary mode. Wire it as the default dev-mode and single-binary cache backend. Redis remains available for production.

Likely paths:
- framework/core/interfaces.go
- container.go
- config/config.go

Verify commands:
- go test ./... -count=1 -run Cache
- docket validate $TICKET_ID

Out of scope:
- Changes to the Redis cache adapter or production multi-process path.
- Broad framework refactors outside the cache adapter surface.

## Acceptance Criteria
- [ ] ship dev starts and performs cache get/set/delete without Redis running
- [ ] Otter adapter implements the Cache interface: Get, Set, Delete, Flush
- [ ] Config CACHE_DRIVER=memory selects Otter; CACHE_DRIVER=redis selects Redis adapter
- [ ] go test ./... -count=1 -run Cache passes including in-memory adapter tests
