# TKT-113: A09 - Signed Managed Hooks For Backup, Restore, And Runtime Status

## Description
**Source:** `docs/architecture/09-standalone-and-managed-mode.md`
**Roadmap Group:** Managed bridge
**Original Status:** `[ ] todo`

**Status:** `[ ] todo`
**Files:** `app/router.go`, `app/web/controllers/` (new managed endpoints), `framework/security/` or equivalent signing helpers, tests

**Context:**
The control plane should manage hosted apps through stable signed hooks, not arbitrary repo edits.
The runtime needs a narrow bridge for backup/restore orchestration and effective-status reporting.

**What to do:**
1. Define signed managed endpoints or equivalent hooks for:
   - runtime metadata/status
   - backup initiation
   - restore initiation
2. Add replay protection and shared-secret verification.
3. Keep the contract narrow and backend-agnostic.
4. Add tests for signature verification and unauthorized access.

**Done when:** A managed GoShip app exposes a minimal signed bridge for status, backup, and restore
without coupling the runtime to any one control-plane implementation.
