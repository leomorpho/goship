# TKT-605: Scaffold layout shell and .env entry from UIProvider in ship new

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-605` before committing. -->

## Description
Objective: Make ship new write a provider-aware base layout templ, a UI_PROVIDER .env entry, and a UI_PROVIDER line in .env.example so the scaffolded app loads the correct CSS/JS assets from the start.

Context: Depends on TKT-603 (UIProvider in NewProjectOptions). The layout is baked at scaffold time — no runtime switching. Three providers:
- franken: injects UIkit 3 CDN script + FrankenUI CSS link in <head>
- daisy: injects Flowbite CDN script in <head>; DaisyUI ships via Tailwind so no separate CDN needed
- bare: no external asset tags injected

Likely files:
- tools/cli/ship/internal/commands/project_new_layout.go
  - renderBaseLayoutTempl(provider string) string — returns a minimal app/views/web/layouts/base.templ content with the correct <head> asset block per provider; function switches on provider constant
- tools/cli/ship/internal/commands/project_new.go
  - baseScaffoldFiles(opts): add app/views/web/layouts/base.templ using renderBaseLayoutTempl(opts.UIProvider)
  - renderStarterDotEnv(opts): append UI_PROVIDER=<opts.UIProvider> line
  - renderStarterDotEnvExample(): add UI_PROVIDER=franken line with a comment explaining the three valid values
  - canonicalGeneratedProjectFiles in project_new_layout.go: add app/views/web/layouts/base.templ
- tools/cli/ship/internal/commands/project_new_layout_test.go (new file)
  - TestRenderBaseLayoutTempl: table-driven over all three providers, assert correct CDN tags present/absent

Likely symbols:
- renderBaseLayoutTempl (new func)

- renderStarterDotEnv (updated to accept opts)
