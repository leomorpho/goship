## Description
After the foundation batteries and notification reset are underway, the remaining retained first-party batteries need to converge on the same standalone battery contract instead of each keeping its own custom module shape.

Scope:
- i18n
- flags
- auditlog
- pwa
- ai
- admin if it remains a first-party battery

Deliver:
1. Align retained supporting batteries with the canonical contract
2. Remove split or custom public surfaces that are no longer justified
3. Ensure docs/tooling present a coherent first-party battery catalog

Likely paths:
- modules/i18n
- modules/flags
- modules/auditlog
- modules/pwa
- modules/ai
- modules/admin

Out of scope: auth/twofactor rebuild and notifications family implementation.

Verify commands:
- go test ./modules/...
- ship verify --profile strict

## Acceptance Criteria
- [ ] retained supporting batteries match the canonical battery contract
- [ ] first-party battery docs/catalog no longer present a mixed set of public shapes
- [ ] go test ./modules/... passes
