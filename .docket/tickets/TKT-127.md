# TKT-127: `ship i18n:init` command (existing apps)

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-127` before committing. -->

## Description
Add an initializer command for projects that were created without i18n enabled.

## Hierarchy
Parent: `TKT-116`
Blocks: `TKT-129`

## Scope
1. Implement `ship i18n:init` command.
2. Scaffold baseline locale files if missing.
3. Add deterministic output describing next migration steps.

## Explicit behavior
1. Must be idempotent.
2. Must not overwrite existing locale files unless `--force` is passed.
3. Must print exact follow-up commands for migration loop.

## Validation
1. Unit tests for success, idempotent rerun, and overwrite guard.
2. CLI dispatch tests for help/usage and exit codes.

## Acceptance Criteria
- [x] Running `ship i18n:init` in a non-i18n app creates baseline locale scaffolding.
- [x] Re-running without `--force` does not clobber existing locale files.
- [x] CLI output includes deterministic next-step commands for LLM workflows.
