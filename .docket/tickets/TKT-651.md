# TKT-651: finish ship module:add 2fa install contract

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-651` before committing. -->

## Description
tools/cli/ship/internal/commands/module.go still defines the 2fa module with raw TODO snippets for both container wiring and auth routes. That means ship's module installer advertises a module it cannot install to a usable state, which is not acceptable for a framework trying to compete on DX.

Fix:
1. Replace the 2fa container snippet with real service wiring
2. Replace the auth router snippet with real route registration via modules/2fa/routes.go
3. Add or update tests so module:add 2fa never emits TODO placeholders

Likely paths:
- tools/cli/ship/internal/commands/module.go
- tools/cli/ship/internal/commands/module_test.go
- modules/2fa/routes.go
- composition/wiring code referenced by the snippet

Out of scope: redesigning the 2FA module itself.

Verify commands:
- go test ./tools/cli/ship/internal/commands/...
- Manual: run ship module:add 2fa in a cleanroom app and inspect the generated wiring

## Acceptance Criteria
- [x] module:add 2fa emits real container wiring instead of TODO placeholders : implemented + verified in this branch
- [x] module:add 2fa emits real auth route registration instead of TODO placeholders : implemented + verified in this branch
- [x] module tests fail if 2fa snippets regress back to TODO text : implemented + verified in this branch
- [x] go test ./tools/cli/ship/internal/commands/... passes : implemented + verified in this branch
