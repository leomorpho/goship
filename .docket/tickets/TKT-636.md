# TKT-636: rename framework/context → framework/appcontext (stdlib name collision)

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-636` before committing. -->

## Description
The package at framework/context is named `context`, colliding with the Go stdlib `context` package. Every importer must alias it. Rename the directory and package declaration to `appcontext` and update all import paths.

Likely paths:
- framework/context/context.go — rename dir to appcontext, change package declaration
- framework/web/middleware/auth.go — main importer
- Any file importing github.com/leomorpho/goship/framework/context

Out of scope: no logic changes, pure rename only.

Verify commands:
- grep -r 'framework/context' --include='*.go' . (should return zero results after)
- go build ./...
- go test ./...

## Acceptance Criteria
- [ ] directory renamed from framework/context to framework/appcontext
- [ ] package declaration changed from `package context` to `package appcontext`
- [ ] all import paths across codebase updated to framework/appcontext
- [ ] no remaining imports of framework/context
- [ ] go build ./... passes
- [ ] go test ./... passes
