# TKT-645: remove context.TODO() from SQL SMS sender bootstrap

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-645` before committing. -->

## Description
modules/notifications/sms_store_sql.go:23 calls aws config.LoadDefaultConfig(context.TODO(), ... ) inside NewSQLSMSSender(). That startup path cannot be cancelled, does not inherit caller deadlines, and sets a bad precedent after the mailer context issue.

Fix:
1. Add a context-aware constructor or move AWS SNS client construction up to the module/bootstrap layer
2. Remove context.TODO() from the SMS sender path
3. Keep the sender behavior unchanged once initialized

Likely paths:
- modules/notifications/sms_store_sql.go
- modules/notifications/module.go
- framework/bootstrap/ or other composition root that constructs the sender

Out of scope: changing SMS delivery behavior, retries, or SNS message formatting.

Verify commands:
- go build ./...
- go test ./modules/notifications/...
