# TKT-589: Generate .env.example in ship new with all required and optional env vars documented

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-589` before committing. -->

## Description
When ship new creates a new app, it must emit a .env.example file that documents every env var the app may need: required vars (APP_KEY, DATABASE_URL, etc.), optional vars with defaults (CACHE_DRIVER=memory, QUEUE_DRIVER=backlite), and battery-specific vars injected on ship module:add (STRIPE_KEY, RESEND_API_KEY, etc.). Include inline comments explaining each var. Table stakes for Rails/Laravel parity.

Likely files:
- tools/cli/ship/internal/commands/new.go
- tools/cli/ship/internal/templates/ (new .env.example template)
- tools/cli/ship/internal/commands/module_add.go (inject battery vars on module:add)

Verify commands:
- go test ./tools/cli/ship/... -count=1 -run New
- docket validate $TICKET_ID

Out of scope:
- Secret rotation or vault integration.
- Dynamic env var discovery at runtime.

## Acceptance Criteria
- [ ] ship new myapp generates a .env.example file in the project root
- [ ] .env.example includes all required vars (APP_KEY, DATABASE_URL) with inline comments
- [ ] .env.example includes optional vars with defaults (CACHE_DRIVER=memory, QUEUE_DRIVER=backlite)
- [ ] ship module:add billing appends STRIPE_KEY, STRIPE_WEBHOOK_SECRET to .env.example
- [ ] go test ./tools/cli/ship/... -count=1 -run New passes
