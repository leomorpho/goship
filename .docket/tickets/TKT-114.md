# TKT-114: A09 - SQLite-First Promotion Path To Postgres

## Description
**Source:** `docs/architecture/09-standalone-and-managed-mode.md`
**Roadmap Group:** Runtime evolution
**Original Status:** `[ ] todo`

**Status:** `[ ] todo`
**Files:** `docs/roadmap/01-framework-plan.md`, `config/config.go`, any new migration/export support packages

**Context:**
The default product path is single-binary SQLite, but some apps will eventually need Postgres.
GoShip should make that promotion possible without building provider-specific assumptions into the runtime.

**What to do:**
1. Define the runtime metadata needed to report current DB mode and migration compatibility.
2. Define the first supported promotion workflow from SQLite to Postgres.
3. Document SQL portability constraints for framework and module authors.
4. Identify the minimum tooling or export/import hooks the framework must expose.

**Done when:** GoShip has a documented and technically grounded promotion contract from SQLite to
Postgres, with enough runtime metadata and portability rules to support a manual first migration path.
