# TKT-667: rebuild auth and two-factor batteries against core seams instead of app-domain modules

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-667` before committing. -->

## Description
Auth and 2FA are useful batteries, but the current shape is too coupled to profile, notifications, onboarding, and legacy helper packages. This ticket rebuilds them on top of core seams after the battery contract and domain ejection work are established.

Deliver:
1. Auth becomes a clean installable battery with explicit core seam dependencies
2. Two-factor becomes either an auth add-on battery or a clearly bounded sibling battery
3. authsupport is folded into the rebuilt auth stack or reduced to internal support code
4. Generator/tooling support aligns with the rebuilt auth/2FA contracts

Likely paths:
- modules/auth
- modules/2fa
- modules/authsupport
- router/bootstrap/module catalog wiring
- ship module:add tooling/tests

Out of scope: preserving the current auth/profile coupling if it conflicts with the new architecture.

Verify commands:
- go test ./modules/auth/...
- go test ./modules/2fa/...
- ship verify --profile strict

## Acceptance Criteria
- [ ] auth matches the canonical battery contract and depends on core seams instead of app-domain modules
- [ ] two-factor has a clear battery boundary relative to auth
- [ ] authsupport is no longer a public pseudo-module surface
- [ ] ship module tooling/docs align with the rebuilt auth/2FA batteries
- [ ] ship verify --profile strict passes
