# TKT-221: Interop: add verify preflight for contract mismatches

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-221` before committing. -->

## Description
The runtime already emits machine-readable metadata and handshake details, but orchestration-facing
flows still lack one canonical preflight gate. This ticket is for adding a `ship verify` preflight
that rejects unsupported deploy/upgrade/promote combinations before orchestration begins.
