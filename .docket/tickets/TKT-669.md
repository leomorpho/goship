# TKT-669: Epic: frontend contract surface for templ islands and SvelteKit

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-669` before committing. -->

## Description
Objective: Epic: frontend contract surface for templ islands and SvelteKit.

Context:
GoShip should not force one rendering strategy. The framework should own the backend contract surface while supporting two explicit frontend lanes: templ plus islands as the default GoShip UI story, and SvelteKit-first as the blessed split-frontend consumer. This epic defines the next phase after the initial API-only and SvelteKit contract work: make the backend contract more discoverable, more machine-readable, and more usable by humans and LLMs across both lanes.

Rails/Laravel DX bar:
- Keep one obvious default UI lane inside GoShip: templ plus route-local islands for richer interaction.
- Keep one obvious split-frontend lane outside GoShip: SvelteKit-first consuming a blessed same-origin contract.
- Put framework value in backend contracts, generators, auth/session semantics, and runtime seams rather than coupling product identity to HTMX.
