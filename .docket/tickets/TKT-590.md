# TKT-590: Validate required env vars are present when running ship module:add for a battery

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-590` before committing. -->

## Description
When ship module:add <battery> is run, check that all required environment variables for that battery are present in .env or the shell. If missing, emit a clear warning listing the missing vars with descriptions and where to get them. Do not abort the installation — emit the warning and continue so developers can configure later. Example: ship module:add billing should warn if STRIPE_KEY and STRIPE_WEBHOOK_SECRET are not set.

Likely files:
- tools/cli/ship/internal/commands/module_add.go
- tools/cli/ship/internal/templates/ (per-battery env var manifest)

Verify commands:
- go test ./tools/cli/ship/... -count=1 -run ModuleAdd
- docket validate $TICKET_ID

Out of scope:
- Runtime config validation (that is ship config:validate).
- Secrets management or vault integration.

## Acceptance Criteria
- [ ] ship module:add billing warns if STRIPE_KEY is not set in .env or environment, with a message showing the exact var name and where to get it
- [ ] Warning is emitted but installation still proceeds (non-blocking)
- [ ] Each battery has a manifest listing its required env vars
- [ ] go test ./tools/cli/ship/... -count=1 -run ModuleAdd passes
