# TKT-617: Add runtime-report and stable access-classification tools to ship-mcp for stronger agent context.

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-617` before committing. -->

## Description
Objective: Add runtime-report and stable access-classification tools to ship-mcp for stronger agent context.

Context:
The MCP server exposes routes and modules, but it still lacks a direct runtime-report tool and its current route filtering cannot distinguish admin routes properly. Agents need one reliable structured surface for runtime facts and route access levels.

Rails/Laravel DX bar:
- Agent tooling should expose the same trustworthy facts a strong human operator would reach for first.
- Filters that silently do nothing are worse than no filter at all.

Likely paths:
- tools/mcp/ship/internal/server/tools.go
- tools/mcp/ship/internal/server/tools_ship.go
- tools/mcp/ship/internal/server/tools_test.go
- tools/cli/ship/internal/commands/describe.go
- tools/cli/ship/internal/commands/describe*_test.go
- tools/cli/ship/internal/commands/runtime_report.go

Likely symbols:
- `ship_runtime_report`
- route access classification (`public`, `auth`, `admin`)
- `ship describe` JSON payload

Tests to add or update:
- MCP tests for new runtime-report tool and route classification behavior.
- CLI describe contract tests if the JSON schema must grow to support stable access classification.

Verify commands:
- docket validate TKT-613
- go test ./tools/mcp/ship/... ./tools/cli/ship/internal/commands -count=1

Out of scope:
- General MCP redesign beyond these context surfaces.
- Framework runtime behavior unrelated to reporting and route inventory.
- Hosted-control-plane-only tooling.

## Acceptance Criteria
