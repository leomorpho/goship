# TKT-647: codify auth redirect and logout session semantics

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-647` before committing. -->

## Description
There are unresolved TODOs around authentication/session behavior that are framework-facing, not app-specific.

Confirmed issues:
- modules/authsupport/auth.go:84 manually expires the auth cookie with a TODO noting the behavior differs from the vanilla starter and the root cause is unknown
- framework/web/middleware/auth_test.go:54 does not assert the unauthenticated redirect semantics beyond 'no error'
- framework/web/middleware/auth.go still carries stale commentary around redirect behavior that should be backed by tests instead of comments

Fix:
1. Decide and document the canonical unauthenticated behavior for RequireAuthentication()/RequireNoAuthentication()
2. Add tests that assert status code, redirect target, and session side effects
3. Audit Logout() and either keep the explicit cookie expiry with tests proving it is needed or simplify it if redundant
4. Remove stale TODO/workaround commentary once behavior is covered

Likely paths:
- modules/authsupport/auth.go
- framework/web/middleware/auth.go
- framework/web/middleware/auth_test.go

Out of scope: profile photo caching in auth middleware, broader auth feature changes.

Verify commands:
- go test ./framework/web/middleware/...
- go test ./modules/authsupport/...

## Acceptance Criteria
- [ ] RequireAuthentication()/RequireNoAuthentication() behavior is specified by tests including status code and redirect target
- [ ] Logout() session-clearing behavior is covered by tests
- [ ] explicit cookie expiry in Logout() is either proven necessary or removed
- [ ] stale auth TODO/workaround comments removed or replaced with explanatory comments backed by tests
- [ ] go test ./framework/web/middleware/... passes
- [ ] go test ./modules/authsupport/... passes
