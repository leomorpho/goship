# TKT-601: Epic: Pluggable UI Provider System

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-601` before committing. -->

## Description
Establish the UI provider pattern for GoShip so generated apps can target different UI libraries (FrankenUI, DaisyUI, bare Tailwind) at project-init time.

Scope contract:
- ship new is the only thing that is provider-aware.
- Generated templ files emit bare structural HTML + HTMX wiring only — no library-specific classes.
- The provider drives: (a) which CSS/JS assets the layout head injects, and (b) the UI_PROVIDER entry in .env.
- Once the user edits generated code, it is their responsibility. No conversion support is promised.
- Admin and other self-contained modules are explicitly out of scope.

Likely paths:
- config/config_schema.go
- tools/cli/ship/internal/commands/project_new.go
- tools/cli/ship/internal/commands/project_new_layout.go
- tools/cli/ship/internal/templates/starter/testdata/scaffold/app/views/web/pages/
