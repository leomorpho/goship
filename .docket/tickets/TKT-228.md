# TKT-228: Interop: make ship upgrade emit readiness and blocker JSON

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-228` before committing. -->

## Description
`ship upgrade` currently mutates the Goose pin directly but only promises the readiness/blocker
contract in docs. This ticket is for making upgrade preflight explicit: emit a machine-readable
readiness report with blocking conditions and remediation hints before or alongside mutation.
