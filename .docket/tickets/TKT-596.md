# TKT-596: [Flags Layer C] Add code-default fallback to Service.Enabled()

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-596` before committing. -->

## Description
Objective: When the DB or cache is unavailable during a flag lookup, fall back to the hardcoded default from the registry instead of returning an error that crashes the call site.

Context:
Currently `Service.Enabled()` returns `(false, err)` if the DB lookup fails. Callers that don't handle the error carefully will gate users out of features during a transient DB hiccup. With the registry in place (TKT-594), we know the intended default — so we should use it.

Resolution order (matching the JS app doc):
1. **Cache hit** → return cached value immediately.
2. **DB hit** → evaluate flag (global enabled + user targeting + rollout) and cache result.
3. **DB/cache miss or error** → consult `flags.Lookup(key)` for the `FlagDefinition.Default`. Return the default and log a WARN with the original error so it's observable.

This fallback must be transparent at the call site — `Enabled()` still returns `(bool, error)`. The error is still returned so callers can distinguish "answered from fallback" if they care, but the bool is always a safe answer.

Likely files:
- modules/flags/service.go — update `lookup()` to call `flags.Lookup(key)` on error and return default
- modules/flags/service_test.go — add fallback-specific test cases

Likely symbols:
- Service.lookup
- Service.Enabled
- flags.Lookup

Tests to add or update:
- modules/flags/service_test.go:
  - TestEnabled_FallsBackToDefault_OnStoreError: store returns error → Enabled returns definition.Default
  - TestEnabled_FallsBackToDefault_DefaultFalse: definition.Default=false → returns false on store error
  - TestEnabled_FallsBackToDefault_DefaultTrue: definition.Default=true → returns true on store error
  - TestEnabled_FallsBackToDefault_UnknownKey: no registry entry → returns false (safe default) and error
  - TestEnabled_NoFallback_OnSuccess: store succeeds → fallback code path is not exercised
