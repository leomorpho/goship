# TKT-622: Publish the canonical SvelteKit integration guide for GoShip API-only apps.

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-622` before committing. -->

## Description
Objective: Publish the canonical SvelteKit integration guide for GoShip API-only apps.

Context:
If GoShip supports one blessed custom-frontend path, it should be documented explicitly. This guide should explain how a SvelteKit frontend talks to a GoShip API-only backend without hand-wavy auth, CSRF, session, cookie, or deployment assumptions.

Rails/Laravel DX bar:
- The supported custom-frontend story should be concrete enough that a weaker LLM can follow it without inventing architecture.
- Keep scope narrow: document the one path we intend to support well.

Likely paths:
- docs/guides/
- docs/guides/08-building-an-api.md
- docs/00-index.md
- docs/reference/01-cli.md
- README.md

Likely symbols:
- SvelteKit same-origin integration
- API-only starter mode
- auth/session/CSRF contract
- deployment and local-dev topology notes

Tests to add or update:
- Doc-sync coverage if this guide becomes part of the canonical docs set.
- Any example-snippet validation added for the guide.

Verify commands:
- docket validate TKT-618
- go test ./tools/cli/ship/internal/commands -count=1
- sed -n '1,260p' docs/guides/08-building-an-api.md

Out of scope:
- Implementing a SvelteKit code generator.
- Supporting non-SvelteKit frontend frameworks.
- OpenAPI/spec generation.

## Acceptance Criteria
- [ ] Code or docs changes stay limited to the described scope.
