# TKT-182: Move bridge adapters out of app/foundation and deduplicate jobs wiring

## Description
Move adapter bridges out of `app/foundation` and deduplicate jobs wiring across `cmd/web` and `cmd/worker`.

Context:
Core adapter bridges and module bridges in app/foundation are reusable and currently duplicated in runtime wiring.

Scope:
- Relocate bridges to framework/module ownership.
- Create one shared jobs wiring path consumed by both cmds.

Out of scope:
- Replacing jobs backend choices.

Likely paths:
- `app/foundation/core_*adapter*.go`
- `app/foundation/*bridge*.go`
- `cmd/web/main.go`
- `cmd/worker/main.go`

Verify commands:
- `go test ./app/foundation/... ./cmd/web ./cmd/worker`
- `go test ./...`

## Acceptance Criteria
- [ ] Core/module bridge adapters no longer live in app/foundation.
- [ ] cmd/web and cmd/worker share one jobs wiring path.
- [ ] Adapter and jobs wiring tests cover moved logic.
