# TKT-604: Strip starter scaffold templ pages to bare structural HTML with HTMX wiring only

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-604` before committing. -->

## Description
Objective: Remove all UI-library-specific class names and styling from the starter scaffold templ files so generated output is purely structural HTML with HTMX attributes. Also gut the starter CSS functions since those classes will no longer exist.

Context: This is independent of the provider flag work. The scaffold pages currently use starter-* classes that are tightly coupled to the starter CSS bundle. The principle is: generators own the wiring (routes, data binding, HTMX plumbing, data-* slots), not the styling. Once a developer has these files, they dress them up with whatever library they chose.

Likely files:
- tools/cli/ship/internal/templates/starter/testdata/scaffold/app/views/web/pages/landing.templ
- tools/cli/ship/internal/templates/starter/testdata/scaffold/app/views/web/pages/home_feed.templ
- tools/cli/ship/internal/templates/starter/testdata/scaffold/app/views/web/pages/profile.templ
- Corresponding *_templ.go generated files must be re-generated after .templ edits (run: templ generate)
- tools/cli/ship/internal/commands/project_new.go
  - renderStarterStylesSource(): replace starter-* variable block with an empty comment placeholder so the file exists but ships no opinions
  - renderStarterStylesBundle(): return an empty string; the pre-built bundle is dead once starter-* classes are gone

Rules for the stripped templ output:
- Keep: data-component, data-slot, data-action attributes (discovery/testing hooks)
- Keep: hx-* attributes (HTMX wiring)
- Keep: structural HTML tags (<section>, <form>, <input>, <button>, <h1>, etc.)
- Remove: all class= attributes containing library or starter-specific tokens
- Minimal class= is allowed only for layout structure where absolutely unambiguous (e.g. class=hidden for sr-only)

Likely symbols:
- Landing (templ component)
- HomeFeed (templ component)
- ProfilePage (templ component)
- renderStarterStylesSource (updated)
- renderStarterStylesBundle (updated)

Tests to add or update:
- tools/cli/ship/internal/commands/project_new_test.go or a dedicated scaffold_pages_contract_test.go: assert that no starter-* class appears in generated scaffold output; assert styles_bundle.css is empty

Verify commands:
- docket validate TKT-604

Out of scope:
- Changing the layout shell or asset injection (TKT-605).
- Removing data-* attributes.
