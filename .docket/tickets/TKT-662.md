# TKT-662: rebuild foundation batteries as the canonical first-party reference set

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-662` before committing. -->

## Description
The first implementation wave should prove the new battery contract on the cleanest, highest-value batteries before tackling the more entangled surfaces. The canonical reference set is jobs, storage, emailsubscriptions, and paidsubscriptions.

Deliver:
1. Make these batteries conform to the canonical standalone battery contract
2. Remove split or inconsistent public surfaces where possible
3. Ensure generator/module tooling treats them as the model batteries
4. Update docs/tests so these are the reference examples for first-party installable batteries

Likely paths:
- modules/jobs
- modules/storage
- modules/emailsubscriptions
- modules/paidsubscriptions
- tools/cli/ship/internal/commands/module.go and related tests/docs

Out of scope: notifications, auth, profile-domain extraction.

Verify commands:
- go test ./modules/jobs/...
- go test ./modules/storage/...
- go test ./modules/emailsubscriptions/...
- go test ./modules/paidsubscriptions/...
- ship verify --profile strict
