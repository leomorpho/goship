# TKT-153: Generate typed i18n keys for Go and JS to reduce string drift

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-153` before committing. -->

## Description
Introduce generated typed key artifacts so application code avoids raw ad-hoc i18n key strings where possible.

## Acceptance Criteria
- [ ] Generated typed keys exist for both Go and JS/TS surfaces.
- [ ] Key generation is deterministic and CI-friendly.
- [ ] Docs specify standard usage and allowed escape hatches.
