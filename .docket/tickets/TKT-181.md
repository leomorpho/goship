# TKT-181: Split app/foundation into minimal composition and extracted runtime bootstrap

## Description
Split `app/foundation` into minimal app composition and extracted reusable runtime bootstrap packages.

Context:
`app/foundation/container.go` mixes app composition with reusable infra/bootstrap logic.

Scope:
- Keep app composition in app layer.
- Extract reusable runtime concerns to framework-owned packages.

Out of scope:
- Feature additions during split.

Likely paths:
- `app/foundation/container.go`
- `app/foundation/container_migrations.go`
- `app/foundation/sqlite_connection.go`
- `framework/**` (new runtime packages)

Verify commands:
- `go test ./app/foundation/... ./framework/...`
- `go test ./cmd/web ./cmd/worker`

## Acceptance Criteria
- [ ] Reusable runtime/bootstrap concerns are extracted from app/foundation.
- [ ] app/foundation remains composition-focused and materially smaller.
- [ ] Container startup/shutdown behavior remains functionally equivalent for supported modes.
