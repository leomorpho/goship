<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-609` before committing. -->

## Description
Objective: Epic: P12-E4 External Frontend Contract

Context:
This epic defines the fastest supported path for teams who want GoShip as a REST-first backend with a custom frontend. The contract must stay convention-first: one blessed split-frontend story, clear auth/runtime boundaries, and tooling that remains friendlier to LLMs than ad hoc project-by-project glue.

Rails/Laravel DX bar:
- Prefer one obvious supported split-frontend path instead of vague framework neutrality.
- Keep the backend-first happy path stronger than custom integration, but make the blessed custom path coherent and well-documented.

Likely paths:
- .docket/tickets/
- docs/roadmap/01-framework-plan.md
- docs/guides/08-building-an-api.md
- docs/guides/
- tools/cli/ship/internal/commands/project_new.go
- tools/cli/ship/internal/templates/starter/
- tools/cli/ship/internal/policies/
- framework/api/
- framework/testutil/

Likely symbols:
- Child leaf ticket titles under this coordination ticket are the executable work units.
- Parent and blocked_by relationships in Docket are the source of execution order.
- The supported split-frontend story must stay explicit, same-origin first, and narrow enough that humans and LLMs can apply it reliably.

Tests to add or update:
- Child leaves under this epic should add or update generator, doctor, verify, API, and testutil contract suites when behavior changes.
- Docket topology for child leaves should remain valid after grooming or reordering.
- Validation should pass for this coordination ticket and its touched child tickets.

Verify commands:
- docket validate TKT-615
- docket list --state open --label roadmap:production-system
- go test ./tools/cli/ship/internal/commands ./framework/api ./framework/testutil/... -count=1

Out of scope:
- Supporting arbitrary frontend generators or every JS framework under the sun.
- Reintroducing framework-owned OpenAPI generation as the core contract surface.
- Weakening the templ-first default in order to make the split-frontend path possible.
