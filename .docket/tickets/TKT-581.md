# TKT-581: Implement ship module:remove command to reverse ship module:add deterministically

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-581` before committing. -->

## Description
Implement ship module:remove <module-name> that reverses what ship module:add did: removes injected marker comment blocks from container.go and router.go, removes module registration, and cleans up generated files. The command must be idempotent — running it twice must produce the same result as running it once.

Likely paths:
- tools/cli/ship/internal/commands/module_remove.go (new)
- tools/cli/ship/internal/commands/module_add.go (reference)

Verify commands:
- go test ./tools/cli/ship/... -count=1 -run Module
- docket validate $TICKET_ID

Out of scope:
- Rolling back database migrations (migrations are not reversible in this command).
- Changes to ship module:add itself unless required for symmetry.

## Acceptance Criteria
- [ ] ship module:remove <name> cleanly undoes ship module:add <name>.
- [ ] Running remove twice is safe and idempotent.
- [ ] Remove of an uninstalled module fails gracefully with a clear message.
