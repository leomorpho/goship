# TKT-653: make embedded test database lifecycle deterministic

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-653` before committing. -->

## Description
framework/bootstrap/container.go:390 has a TODO noting that the embedded database path is not dropped/recreated for test runs unless it is in-memory. That leaves test isolation dependent on configuration details instead of the framework enforcing a clean state.

Fix:
1. Define the canonical lifecycle for embedded test databases
2. Ensure test bootstrapping starts from a clean embedded DB state even when the connection is file-backed
3. Add tests or startup-contract coverage that proves repeated test runs do not leak state

Likely paths:
- framework/bootstrap/container.go
- framework/bootstrap/*_test.go
- config paths affecting embedded test DB connection

Out of scope: changing the non-test embedded database lifecycle.

Verify commands:
- go test ./framework/bootstrap/...
- go test ./...
