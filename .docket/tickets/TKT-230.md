# TKT-230: Interop: emit module adoption metadata from runtime report surfaces

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-230` before committing. -->

## Description
`ship describe --pretty` already exposes a shared-infra adoption summary, but the runtime-facing
surfaces used by future orchestration tooling do not yet carry the same module adoption metadata.
This ticket is for extending the runtime report surfaces so external tooling can inspect module
identity, source, and adoption without parsing repo internals.
