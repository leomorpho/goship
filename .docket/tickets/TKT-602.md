# TKT-602: Add UIConfig with Provider field to config schema

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-602` before committing. -->

## Description
Objective: Add a UIConfig struct to the config schema so the UI provider is a typed, validated first-class config value.

Context: This is the foundational config change. All subsequent UI provider tickets depend on the types and constants defined here.

Likely files:
- config/config_schema.go — add uiprovider type, constants UIProviderFranken/UIProviderDaisy/UIProviderBare, UIConfig struct with Provider uiprovider field
- config/config.go — add UI UIConfig to the Config struct, add applyUIProviderDefaults() called from GetConfig() that sets Provider to UIProviderFranken when empty
- config/config_test.go — test that default resolves to franken, test that an unrecognised provider value returns an error

Likely symbols:
- uiprovider (new type)
- UIProviderFranken, UIProviderDaisy, UIProviderBare (new constants)
- UIConfig (new struct)
- applyUIProviderDefaults (new func in config.go)
- Config.UI (new field)

Tests to add or update:
- config/config_test.go: TestUIProviderDefault, TestUIProviderValidation

Verify commands:
- docket validate TKT-<id>
- go test ./config/... -count=1

Out of scope:
- Any changes to ship new or templ files.
- Any runtime UI switching logic.

## Acceptance Criteria
- [ ] UIConfig struct with Provider field and three provider constants exists in config_schema.go
- [ ] GetConfig() defaults Provider to UIProviderFranken when env var is absent
- [ ] go test ./config/... passes
