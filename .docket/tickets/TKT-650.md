# TKT-650: restore flash message dismissal behavior and session-error policy

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-650` before committing. -->

## Description
The flash-message stack has two framework-level TODOs that point to degraded UX and unclear observability.

Confirmed issues:
- framework/web/components/gen/messages_templ.go carries a TODO stating message dismissal behavior was broken relative to the original Pagoda system
- framework/repos/uxflashmessages/msg.go downgrades session load failures to warning 'for now' because the errors are noisy, but the intended policy is undefined

Fix:
1. Restore reliable dismiss behavior for rendered flash messages
2. Ensure the source of truth for the messages component is maintainable and can be regenerated cleanly if the generated file is committed
3. Decide how session read/write failures for flash messages should be logged and test that policy
4. Remove the TODOs once behavior is explicit

Likely paths:
- framework/web/components/gen/messages_templ.go
- framework/repos/uxflashmessages/msg.go
- source templ/component file for messages if it is missing from the repo

Out of scope: redesigning the visual style of flash messages.

Verify commands:
- go test ./framework/repos/uxflashmessages/...
- go test ./framework/web/...
- Manual: trigger each flash type and dismiss it in the browser

## Acceptance Criteria
- [ ] flash messages dismiss reliably in the rendered UI
- [ ] the messages component has a maintainable source of truth and generated output is in sync
- [ ] flash-message session failures are logged at an intentional level with tests or rationale
- [ ] stale flash-message TODO comments removed
- [ ] go test ./framework/web/... passes
