# TKT-145: Canonical i18n enforcement scope policy

## Description
Define an explicit boundary for which user-facing strings must be i18n-backed and how each surface is enforced.

## Hierarchy
Parent: `TKT-143`
Blocks: `TKT-146`, `TKT-151`, `TKT-152`, `TKT-153`, `TKT-154`, `TKT-155`

## Scope
1. Define mandatory i18n coverage matrix across Go handlers, templ HTML, islands JS, API human messages, and email templates.
2. Define exclusions (logs, metrics keys, SQL, internal error details, tests) with rationale.
3. Map each surface to existing command/policy enforcement (`scan`, `instrument`, `doctor DX029`, completeness checks).
4. Define temporary gaps as explicit known limitations.

## Acceptance Criteria
- [ ] One policy doc defines mandatory coverage and explicit exclusions.
- [ ] Enforcement mapping is explicit (what is checked, by which command).
- [ ] Policy is linked from CLI/i18n docs so LLMs and humans use the same contract.
