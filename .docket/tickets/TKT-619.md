# TKT-619: Add a doc drift gate for README, LLM.txt, MCP docs, and canonical architecture docs.

## Description
Objective: Add a doc drift gate for README, LLM.txt, MCP docs, and canonical architecture docs.

Context:
GoShip already has doc-sync and canonical-doc wording enforcement, but the highest-value entry docs still drifted. Add one named gate that fails when README, generated LLM artifacts, MCP guidance, and canonical architecture docs fall out of sync.

Rails/Laravel DX bar:
- The public and LLM-facing entry surface should not depend on manual vigilance.
- If docs are part of the product, drift must be a tested failure, not a review suggestion.

Likely paths:
- tools/cli/ship/internal/commands/doc_sync_contract_test.go
- tools/cli/ship/internal/commands/help_contract_test.go
- .github/workflows/test.yml
- Makefile
- README.md
- LLM.txt
- docs/reference/02-mcp.md
- MCP_TOOLS.md
- docs/architecture/*.md

Likely symbols:
- canonical-doc drift gate
- doc-sync contract tests
- LLM artifact refresh workflow

Tests to add or update:
- Named doc drift contract tests covering README, `LLM.txt`, MCP docs, and canonical architecture references.
- CI wiring so the drift gate is visible as a first-class status check.

Verify commands:
- docket validate TKT-611
- go test ./tools/cli/ship/internal/commands -count=1
- make test-doc-sync

Out of scope:
- Rewriting the affected docs themselves beyond minimal fixture updates required by the gate.
- MCP runtime feature work.
- Non-canonical docs outside the chosen source-of-truth set.
