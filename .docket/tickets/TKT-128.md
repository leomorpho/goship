# TKT-128: `ship i18n:scan --format json` diagnostics engine

## Description
Build a deterministic diagnostics scanner for hardcoded user-facing literals and missing i18n-key usage.

## Hierarchy
Parent: `TKT-116`
Blocks: `TKT-129`, `TKT-130`, `TKT-131`

## Scope
1. Add `ship i18n:scan` command.
2. Add JSON output schema with stable issue IDs.
3. Support path filtering (`--paths`) and bounded output (`--limit`).

## Explicit behavior
1. JSON output must be stable across runs for unchanged code.
2. Each issue includes: `id`, `kind`, `severity`, `file`, `line`, `column`, `message`, `suggested_key`, `confidence`.
3. Exit code is non-zero only for scanner/runtime errors, not for findings.

## Validation
1. Unit tests with fixtures for Go/templ/JS scanning.
2. Snapshot-style tests for deterministic JSON ordering.

## Acceptance Criteria
- [ ] `ship i18n:scan --format json` returns machine-consumable deterministic diagnostics.
- [ ] `--paths` and `--limit` work as documented.
- [ ] Issue IDs remain stable between identical runs.
