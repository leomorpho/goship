# TKT-672: generate a versioned TypeScript contract package and make the canonical SvelteKit app consume it

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-672` before committing. -->

## Description
Objective: generate a versioned TypeScript contract package and make the canonical SvelteKit app consume it.

Context:
GoShip already has a thin TypeScript-facing helper and an API-only SvelteKit reference app, but the contract is still too manual. This ticket should turn the richer endpoint metadata into one generated, versioned TypeScript package that humans and LLMs can inspect, and it should prove the result by wiring the canonical SvelteKit reference app to consume that package instead of a handwritten placeholder seam.

Rails/Laravel DX bar:
- The contract package should be the first thing an LLM reaches for when building against GoShip from SvelteKit.
- Generated artifacts should stay small, readable, and deterministic.

Likely paths:
- tools/cli/ship/internal/commands/
- framework/api/
- examples/sveltekit-api-only/
- docs/reference/01-cli.md
- docs/guides/08-building-an-api.md

Likely symbols:
- ship contract generation command
- versioned TypeScript package output
- typed fetch helpers
- generated endpoint definitions and envelopes
- reference-app consumption of generated artifacts

Tests to add or update:
- Generated artifact contract tests and snapshot coverage.
- Reference-app drift tests proving the canonical example consumes the generated package correctly.
- CLI/help/docs contract tests for the generation surface.

Verify commands:
- docket validate TKT-672
- go test ./tools/cli/ship/internal/commands ./framework/api -count=1
- sed -n '1,240p' examples/sveltekit-api-only/src/lib/server/goship-contract.ts

Out of scope:
- Supporting non-TypeScript SDKs.
- Building full frontend generators beyond the reference app.
- Turning GoShip into a generic client-SDK factory.

## Acceptance Criteria
- [ ] GoShip can generate one small versioned TypeScript contract package for the blessed split-frontend lane.
- [ ] The canonical SvelteKit reference app consumes the generated contract package rather than a handwritten placeholder seam.
- [ ] Automated tests freeze artifact shape and catch drift between generation output, framework API semantics, and the reference app.
- [ ] Verification commands and outcomes are recorded in ticket notes or handoff.
