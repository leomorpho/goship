# TKT-115: Deprecate lecho and zerolog in favor of slog

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-115` before committing. -->

## Description
Now that slog is integrated as the primary structured logging solution, lecho and zerolog dependencies should be removed from the project to reduce overhead and simplify the stack. 

**What to do:**
1. Identify all remaining locations using zerolog.
2. Remove zerolog and lecho from go.mod.
3. Update any code still relying on these libraries to use framework/logging.

**Status:** `[x] done`
**Files:** `app/web/middleware/recover_test.go`, `go.mod`, `go.sum`, `docs/roadmap/06-dx-and-infrastructure.md`

## Acceptance Criteria
- [x] No production code paths import `github.com/rs/zerolog` or `github.com/ziflex/lecho/v3`.
- [x] `go.mod` and `go.sum` no longer require zerolog or lecho.
- [x] Logging behavior remains equivalent with `framework/logging` + `slog` across web startup, request logging, and error paths.
