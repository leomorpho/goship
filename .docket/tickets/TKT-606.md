# TKT-606: Prove end-to-end scaffold output for all three UI providers with golden contract tests

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-606` before committing. -->

## Description
Objective: Add a golden contract test that scaffolds a project for each provider and asserts both the layout head content and the absence of UI-library classes in generated templ files.

Context: This is the integration-level proof that TKT-604 (bare pages) and TKT-605 (layout shell) work correctly together across all three providers. A future regression in either should break this test before it reaches users.

Likely files:
- tools/cli/ship/internal/commands/ui_provider_contract_test.go (new file)
  - TestUIProviderScaffoldGolden: table-driven over [franken, daisy, bare]
    - call ScaffoldNewProject with UIProvider=<provider> into a temp dir
    - assert base.templ head block contains expected provider asset tags
    - assert none of the scaffolded .templ files contain class names matching /starter-|uk-|btn |daisyui-/
    - assert .env contains UI_PROVIDER=<provider>
    - assert layout file path app/views/web/layouts/base.templ exists

- ScaffoldNewProject (called in test)
- TestUIProviderScaffoldGolden (new)
