# TKT-150: Harden i18n allowlist contract for stability

## Description
Reduce fragility in `.i18n-allowlist` semantics (currently line-number-sensitive entries) and standardize stable identifiers.

## Hierarchy
Parent: `TKT-143`
Blocks: `TKT-155`, `TKT-146`

## Scope
1. Define canonical allowlist selectors (stable IDs first, optional legacy compatibility path for `path:line`).
2. Add migration/backward-compatibility behavior for existing allowlist files.
3. Add tests for line shifts and refactors that should not invalidate intentional allowlist intent.

## Acceptance Criteria
- [ ] Allowlist supports stable selectors that survive line movement.
- [ ] Legacy allowlist behavior is documented and migration-safe.
- [ ] Doctor strict-mode consumes the hardened contract deterministically.
