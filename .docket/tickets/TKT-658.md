# TKT-658: epic: reset first-party module surface around clean standalone batteries

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-658` before committing. -->

## Description
GoShip's current modules tree is carrying multiple architectural eras at once: some packages are true standalone batteries with their own go.mod files, some are local app/framework subpackages, some split service/runtime from routes into sibling packages, and several app-domain modules still live under the framework module surface. Incremental cleanup will keep preserving historical shape instead of converging on a coherent Laravel/Rails-style product surface.

Decision:
- freeze the current mixed modules tree as legacy reference
- define one canonical battery contract
- rebuild the installable module surface intentionally
- eject app/domain code from the framework battery surface
- replace the notifications monolith with smaller batteries instead of refactoring it in place

Target principles:
1. framework/ keeps only core seams
2. modules/ contains only real installable batteries
3. batteries have one public entrypoint and one canonical runtime/install contract
4. old code is mined selectively; it is not the architectural source of truth

Likely paths:
- modules/
- go.mod + go.work
- tools/cli/ship/internal/commands/module.go and related verify/doctor policies
- router/bootstrap/cmd wiring currently composing first-party modules
- docs/reference and architecture docs describing modules and standalone batteries

Out of scope: preserving historical module shapes for compatibility if they conflict with the new battery contract.

Verify commands:
- ship verify --profile strict
- go test ./tools/cli/ship/internal/...
- go test ./...

## Acceptance Criteria
- [ ] GoShip has a documented keep/rewrite/eject decision for every first-party module candidate
- [ ] the canonical battery contract is defined and enforced by policy/tests
- [ ] the mixed legacy modules tree is no longer treated as the future architecture source of truth
- [ ] the notifications monolith has a replacement plan based on smaller batteries
- [ ] ship verify --profile strict passes after the reset work lands
