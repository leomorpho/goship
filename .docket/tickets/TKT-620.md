# TKT-620: Define the blessed external-frontend contract for GoShip: API-only backend, same-origin auth, and one supported custom-frontend story.

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-620` before committing. -->

## Description
Objective: Define the blessed external-frontend contract for GoShip: API-only backend, same-origin auth, and one supported custom-frontend story.

Context:
GoShip already has JSON API helpers, but it does not yet define a first-class product contract for teams who want the GoShip binary as a backend with a custom frontend. This ticket should make the contract explicit without turning GoShip into a vague frontend-agnostic platform.

Rails/Laravel DX bar:
- Constrain the supported split-frontend story until it is boring and reliable.
- Prefer one strong contract over five half-supported frontend options.

Likely paths:
- docs/roadmap/01-framework-plan.md
- docs/guides/08-building-an-api.md
- docs/architecture/03-project-scope-analysis.md
- docs/architecture/09-standalone-and-managed-mode.md
- docs/reference/01-cli.md

Likely symbols:
- API-only app mode
- same-origin auth/session model
- cookie/CSRF/CORS boundary
- supported custom frontend scope (SvelteKit-first)

Tests to add or update:
