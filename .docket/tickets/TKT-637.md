# TKT-637: rename framework/pubsub and framework/repos/pubsub to remove name ambiguity

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-637` before committing. -->

## Description
Two packages share the name `pubsub` but do entirely different things:
- framework/pubsub: in-process raw-bytes broker, implements core.PubSub
- framework/repos/pubsub: SSE-facing adapters (NATS, Redis, InProc) with SSEEvent payloads for browser streaming

Proposed renames:
- framework/pubsub → framework/inprocbroker
- framework/repos/pubsub → framework/repos/ssepubsub

Likely paths:
- framework/pubsub/ (rename dir + package declaration)
- framework/repos/pubsub/ (rename dir + package declaration)
- framework/bootstrap/ (wires both)
- modules/notifications/module.go (imports repos/pubsub)

Out of scope: no logic changes, pure rename only.

Verify commands:
- go build ./...
- go test ./...
