# TKT-621: Implement a blessed API-only ship new starter mode and support it in doctor, verify, and golden-path tests.

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-621` before committing. -->

## Description
Objective: Implement a blessed API-only ship new starter mode and support it in doctor, verify, and golden-path tests.

Context:
Once the external-frontend contract is defined, GoShip should provide one canonical starter mode for backend-only apps. That mode must be a first-class generated-app path, not a hand-edited escape hatch, and it must work with doctor/verify instead of fighting repo-shape assumptions.

Rails/Laravel DX bar:
- The happy path for an API-only GoShip app should be just as deliberate as the templ-first default.
- Supported alternate app modes should be explicit and testable, not folklore.

Likely paths:
- tools/cli/ship/internal/commands/project_new.go
- tools/cli/ship/internal/commands/project_new*_test.go
- tools/cli/ship/internal/templates/starter/
- tools/cli/ship/internal/policies/doctor_repo_checks.go
- tools/cli/ship/internal/commands/verify.go
- docs/guides/01-getting-started.md
- tests/e2e/

Likely symbols:
- `ship new` mode flag/contract
- generated API-only scaffold layout
- doctor/verify starter compatibility

Tests to add or update:
- generator tests for API-only scaffold output.
- doctor/verify tests proving the blessed API-only starter passes the canonical loop.
- Any fresh-app or golden-path tests needed for the new starter mode.

Verify commands:
- docket validate TKT-617
- go test ./tools/cli/ship/internal/commands ./tools/cli/ship/internal/policies -count=1

Out of scope:
- SvelteKit guide content beyond minimal starter references.
- Arbitrary frontend generators.
- Changing the framework repo's own canonical top-level shape.
