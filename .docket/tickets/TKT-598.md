# TKT-598: [Flags Admin UI] Show constant key and code-defined default in flags list

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-598` before committing. -->

## Description
Objective: Update the admin flags list at `/admin/flags` to display the typed constant key and hardcoded default value from the registry alongside the live DB toggle state.

Context:
Currently the admin UI shows only what's in the DB (key string, enabled toggle, description). After TKT-594 introduces the registry, the admin gains a richer picture: the constant name the developer uses in code, the description from the definition, and the default the flag falls back to if the DB is cleared. This makes the admin UI the single place an operator can understand both the runtime state and the code intent.

UI changes per flag row:
- **Constant key**: display the `FlagKey` value (e.g. `ai.enabled`) in a monospace badge — this is what developers use in code.
- **Code default**: show the hardcoded default (On/Off badge) next to the DB toggle — makes it obvious if the live state differs from the default.
- **Description**: already shown; now sourced from the registry definition (synced by TKT-595) rather than manually entered.

For flags that exist in DB but are NOT in the registry (orphans), see TKT-599.

Likely files:
- modules/admin/routes.go — update flags list handler to join registry data with DB rows
- modules/admin/views/flags.templ (or equivalent) — add constant key badge and default badge columns
- modules/flags/registry.go — Lookup() already available after TKT-594

Likely symbols:
- FlagsListHandler (or equivalent in routes.go)
- FlagRow view model (add ConstantKey, CodeDefault fields)
- flags.Lookup
- flags.All

Tests to add or update:
- modules/admin/routes_test.go (or equivalent):
  - TestFlagsListHandler_ShowsRegisteredConstantKey: response HTML contains the expected FlagKey value
  - TestFlagsListHandler_ShowsCodeDefault: response HTML contains correct default badge (On/Off)
  - TestFlagsListHandler_MergesRegistryAndDB: DB row with matching registry entry renders both

Verify commands:
- docket validate TKT-598
- go test ./modules/admin/... -run TestFlagsList -count=1
- go test ./modules/admin/... -count=1

Out of scope:
- Orphaned DB flag detection and display (TKT-599).
- Editing flag definitions from the UI (code is the source of truth).
- Capabilities API (TKT-597).
