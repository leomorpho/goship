# TKT-657: restore ctrl/cmd-click new-tab behavior for HTMX navigation

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-657` before committing. -->

## Description
framework/web/components/gen/core_templ.go contains a broken TODO-marked attempt to make ctrl/cmd-click open HTMX links in a new tab. The current generated script is malformed and the behavior is still not working, which makes the framework's navigation feel non-native.

Fix:
1. Reintroduce ctrl/cmd-click support for HTMX-driven navigation in a valid, tested way
2. Ensure the source of truth for the script is editable and regenerated cleanly if the generated file is committed
3. Add a focused browser or JS-level test if feasible

Likely paths:
- framework/web/components/gen/core_templ.go
- source templ/component/script file that generates the core HTMX helpers
- any frontend test coverage for navigation helpers

Out of scope: replacing HTMX navigation wholesale.

Verify commands:
- go test ./framework/web/...
- Manual: ctrl/cmd-click an HTMX navigation target and confirm it opens a new tab

## Acceptance Criteria
- [ ] ctrl/cmd-click on HTMX navigation targets opens a new tab as expected
- [ ] the generated HTMX helper script is valid and sourced from maintainable code
- [ ] the stale TODO in the generated helper is removed
- [ ] go test ./framework/web/... passes
