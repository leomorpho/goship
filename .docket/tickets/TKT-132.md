# TKT-132: Strict-mode policy integration (`off|warn|error`)

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-132` before committing. -->

## Description
Wire scanner results into `ship doctor` and config-driven strictness policy.

## Hierarchy
Parent: `TKT-117`
Blocks: `TKT-138`, `TKT-123`

## Scope
1. Add strict mode config values: `off`, `warn`, `error`.
2. Integrate i18n findings into doctor output (text + machine format).
3. Add allowlist mechanism for intentional exceptions.

## Explicit behavior
1. `off`: no i18n enforcement findings.
2. `warn`: findings printed, non-blocking exit.
3. `error`: findings fail the command.

## Validation
1. Policy-mode tests for all three modes.
2. Allowlist tests proving scoped suppression.

## Acceptance Criteria
- [x] Strict mode behavior is deterministic and tested.
- [x] `ship doctor` can enforce i18n policy in CI.
