# TKT-112: A09 - Managed Mode Settings Lock And Read-Only Admin Surfaces

## Description
**Source:** `docs/architecture/09-standalone-and-managed-mode.md`
**Roadmap Group:** Framework capability
**Original Status:** `[ ] todo`

**Status:** `[x] done`
**Files:** `config/managed_settings.go`, `app/web/controllers/preferences.go`, `app/views/web/pages/preferences.templ`, `modules/admin/routes.go`, related tests and docs

**Context:**
GoShip should have real standalone settings/admin capability, but managed apps need certain
settings to become read-only or hidden locally so authority can live outside the app.

**What to do:**
1. Define how a setting is marked locally editable, read-only, or externally managed.
2. Expose that state through the app's settings/admin surface.
3. Show externally managed values as managed rather than silently disabling them.
4. Add tests for permission and rendering behavior where practical.

**Done when:** A managed app can present settings as externally managed without removing the
underlying standalone capability from GoShip.
