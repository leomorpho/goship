# TKT-670: define the official frontend lanes and the v1 backend contract boundary

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-670` before committing. -->

## Description
Objective: define the official frontend lanes and the v1 backend contract boundary.

Context:
GoShip's frontend story is still too easy to misread as 'HTMX is the framework' instead of 'GoShip owns backend contracts and supports explicit rendering lanes.' This ticket makes the product boundary explicit: templ plus islands is the default in-framework UI lane, and SvelteKit-first remains the one blessed split-frontend lane. It should also define the minimum v1 backend contract surface that both humans and LLMs can rely on before code generation grows further.

Rails/Laravel DX bar:
- Keep the default GoShip UI lane obvious and opinionated.
- Keep the split-frontend lane narrow and explicit.
- Make the stable seam the backend contract surface, not the frontend technology choice.

Likely paths:
- docs/roadmap/01-framework-plan.md
- docs/guides/08-building-an-api.md
- docs/reference/01-cli.md

Likely symbols:

Tests to add or update:
