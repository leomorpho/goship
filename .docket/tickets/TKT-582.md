# TKT-582: Add fixture test proving ship new + ship module:add + ship module:remove round-trips cleanly

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-582` before committing. -->

## Description
Add an integration fixture test that: (1) runs ship new in a temp dir, (2) runs ship module:add for a battery, (3) verifies the module is installed, (4) runs ship module:remove, (5) verifies the app is back to its original state. This is the round-trip contract for the generator surface.

Likely paths:
- tools/cli/ship/internal/commands/
- or a new e2e fixture test file in tools/cli/ship/

Verify commands:
- go test ./tools/cli/ship/... -count=1 -run RoundTrip
- docket validate $TICKET_ID

Out of scope:
- Testing every battery module in the round-trip (one representative battery is sufficient).
- Browser or Playwright coverage for this ticket.

## Acceptance Criteria
- [ ] Fixture test runs ship new → module:add → module:remove in a temp directory.
- [ ] After remove
- [ ] the generated app tree matches the post-ship-new state.
- [ ] Test runs in CI without Docker.
