# TKT-644: propagate context through mailer — replace context.TODO() in resend driver

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-644` before committing. -->

## Description
framework/repos/mailer/resend.go:45 uses context.TODO() when calling the Resend SDK's SendWithContext(). This means the email send cannot be cancelled, does not inherit request deadlines, and drops any tracing spans from the caller.

The core.Mailer interface's Send() method currently takes no context. Fix:
1. Add ctx context.Context as first parameter to core.Mailer.Send() interface
2. Update all implementations (ResendMailClient, SMTPMailClient, LogMailClient) to accept and pass through context
3. Update all call sites

Likely paths:
- framework/core/interfaces.go (Mailer interface)
- framework/repos/mailer/resend.go
- framework/repos/mailer/smtp.go
- framework/repos/mailer/log.go
- All modules that call mailer.Send()

Out of scope: retries, rate limiting.

Verify commands:
- go build ./...
- go test ./...
