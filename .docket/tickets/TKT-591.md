# TKT-591: Declare and enforce inter-module prerequisites so incompatible battery combinations fail fast

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-591` before committing. -->

## Description
Modules can have dependencies on other modules. For example: billing requires auth (needs a user model and session), notifications may require jobs (async delivery). ship module:add should check these prerequisites before installing and fail with a clear message if a required module is not installed. Each module should declare its dependencies in a machine-readable manifest.

Example:
  ship module:add billing
  Error: billing requires the auth module. Run: ship module:add auth

Likely files:
- tools/cli/ship/internal/commands/module_add.go
- modules/*/module.go (add Requires() []string to module interface)
- tools/cli/ship/internal/policies/ (new prerequisite check)

Verify commands:
- go test ./tools/cli/ship/... -count=1 -run ModuleDep
- docket validate $TICKET_ID

Out of scope:
- Version pinning between modules.
- Transitive dependency resolution (direct deps only for now).
