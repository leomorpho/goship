# TKT-629: Define one canonical project-context bundle for humans and LLMs and lock its schema.

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-629` before committing. -->

## Description
Objective: Define one canonical project-context bundle for humans and LLMs and lock its schema.

Context:
GoShip already exposes `ship describe --json`, `ship doctor --json`, and `ship runtime:report --json`, but weaker models still have to infer which command to run first and how to merge the outputs. Add one stable context bundle surface so both humans and agents can get the important repo, runtime, and next-step facts in one obvious place.

Rails/Laravel DX bar:
The first command an operator or agent reaches for should be obvious and should return trustworthy, task-starting context without extra archaeology.

Likely paths:
- tools/cli/ship/internal/commands/
- tools/cli/ship/internal/cli/
- docs/reference/01-cli.md
- docs/guides/01-ai-agent-guide.md

Likely symbols:
- `ship context --json`
- `context-bundle-v1`
- recommended next commands
- route access classification

Tests to add or update:
- command contract tests for the new context bundle surface
- golden tests for stable machine-readable output
- docs sync for the canonical task-start workflow

Verify commands:
- docket validate TKT-629
- go test ./tools/cli/ship/internal/commands ./tools/cli/ship/internal/cli -count=1

Out of scope:
- Replacing existing command surfaces like `describe`, `doctor`, or `runtime:report`.
- Hosted-control-plane-only reporting.
- Broad agent workflow redesign beyond the canonical context bundle.
