# TKT-577: Add Backlite SQLite job queue adapter and wire it as the default dev/single-binary jobs backend

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-577` before committing. -->

## Description
Add a Backlite (SQLite-backed) job queue adapter to GoShip so the framework can run background jobs without Redis in standalone/single-binary mode. Wire it as the default dev-mode and single-binary jobs backend. Redis/Asynq remains available for production multi-process mode.

Likely paths:
- modules/jobs/
- framework/core/interfaces.go
- container.go
- config/config.go

Verify commands:
- go test ./modules/jobs/... -count=1
- docket validate $TICKET_ID

Out of scope:
- Changes to the Redis/Asynq adapter or production multi-process path.
- Broad framework refactors outside the jobs adapter surface.

## Acceptance Criteria
- [ ] ship dev starts and queues/processes a background job without Redis running
- [ ] Backlite adapter implements the Jobs interface: Enqueue, Process, and inspect queue depth
- [ ] Config QUEUE_DRIVER=backlite selects the adapter; QUEUE_DRIVER=redis selects Asynq
- [ ] go test ./modules/jobs/... -count=1 passes including backlite driver tests
