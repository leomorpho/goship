# TKT-178: Make paidsubscriptions integration key-agnostic

## Description
Make paid subscriptions integration key-agnostic by removing module dependency on app plan constants.

Context:
`modules/paidsubscriptions/routes` imports app-level plan keys, violating module isolation and flexibility.

Scope:
- Replace direct free/pro key assumptions with catalog/service-driven checks.
- Remove module import of `app/subscriptions`.

Out of scope:
- Product pricing changes.

Likely paths:
- `modules/paidsubscriptions/routes/routes.go`
- `cmd/web/main.go`
- `cmd/worker/main.go`

Verify commands:
- `go test ./modules/paidsubscriptions/...`
- `go test ./cmd/web ./cmd/worker`

## Acceptance Criteria
- [ ] Module routes/services no longer import app/subscriptions constants.
- [ ] Plan checks rely on catalog/service semantics
- [ ] not fixed key names.
- [ ] Tests verify custom key compatibility.
