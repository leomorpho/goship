# TKT-624: Expand framework/testutil with JSON, multipart, and SSE assertions for API-heavy apps.

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-624` before committing. -->

## Description
Objective: Expand framework/testutil with JSON, multipart, and SSE assertions for API-heavy apps.

Context:
GoShip already made HTML/form testing easier, but API-heavy and realtime-heavy tests still need too much manual setup. Strengthening `framework/testutil` is a direct speed win for humans and LLMs building JSON or split-frontend apps.

Rails/Laravel DX bar:
- Common test flows should be five-line helpers, not transport boilerplate.
- The default test ergonomics should stay strong even when the app is not HTML-first.

Likely paths:
- framework/testutil/http.go
- framework/testutil/*.go
- framework/testutil/*_test.go
- docs/guides/02-development-workflows.md
- docs/architecture/06-known-gaps-and-risks.md

Likely symbols:
- JSON request helper
- multipart upload helper
- SSE assertion helper
- auth/session convenience helpers

Tests to add or update:
- New testutil unit/integration tests for JSON, multipart, and SSE flows.
- Docs updates showing the canonical helper usage.

Verify commands:
- docket validate TKT-620
- go test ./framework/testutil/... -count=1

Out of scope:
- Full websocket/browser automation abstractions.
- Reworking unrelated controller logic.
- Framework-wide auth changes.

## Acceptance Criteria
