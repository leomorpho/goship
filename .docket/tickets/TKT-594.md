# TKT-594: [Flags Layer A] Define FlagKey typed constant and FlagDefinition registry

<!-- docket: Direct edits are allowed. After editing this ticket, run `docket validate TKT-594` before committing. -->

## Description
Objective: Introduce typed flag constants and a package-level registry so flags are never referenced as raw strings anywhere in the codebase.

Context:
Currently flags are looked up by plain strings (e.g. `svc.Enabled(ctx, "my-flag")`). This ticket replaces that pattern with a typed `FlagKey` alias and a `FlagDefinition` struct that carries the key, human description, and hardcoded default. A package-level `Registry` collects definitions; modules call `flags.Register(...)` in their `module.go` wire-up. This is Layer A — the source of truth that all other layers depend on.

Design:
```go
// FlagKey is a typed string alias — prevents raw string usage at call sites.
type FlagKey string

// FlagDefinition is the code-level declaration of a flag.
type FlagDefinition struct {
    Key         FlagKey
    Description string
    Default     bool
}

// Register adds a definition to the global registry.
// Panics on duplicate key to catch wiring mistakes at startup.
func Register(def FlagDefinition)

// All returns a snapshot of all registered definitions.
func All() []FlagDefinition

// Lookup returns the definition for a key, and whether it was found.
func Lookup(key FlagKey) (FlagDefinition, bool)
```

Application code declares flags as package-level constants:
```go
// In e.g. modules/ai/flags.go:
var FlagAIEnabled = flags.Register(flags.FlagDefinition{
    Key:         "ai.enabled",
    Description: "Enable AI-powered features for this user",
    Default:     false,
})
```

`Service.Enabled()` signature changes from `(ctx, string, ...int64)` to `(ctx, FlagKey, ...int64)` so the compiler enforces typed usage.
